# Money Pace App Navigation Design

## Goal

Money Paceを、縦に情報を並べたダッシュボードから、ホームと管理を明確に分けた個人向け金融アプリへ再構成する。初見では「今月あと使える金額」「今日使える目安」「支出入力」だけを認識でき、管理を選ぶと収入・固定費・予算・分析へ画面遷移する体験にする。

## Chosen Direction

3案を比較した。

- 別URL方式: 明確だが、既存のフォーム送信後の戻り先と状態管理が増える。
- アプリ内遷移方式: 既存APIを維持しつつ、履歴・フォーカス・アニメーションを制御できる。
- モーダル方式: 入力には向くが、履歴や分析の閲覧には狭い。

アプリ内遷移方式を採用する。`#home`と`#manage`をHistory APIで切り替え、対象外のビューは`hidden`にする。JavaScriptが無効でもホームと管理内容へ到達できるマークアップを維持する。

## Home View

- ヘッダー左にブランド、右に「管理」を置く。ホーム表示中は管理だけを主操作として見せる。
- ファーストビューは残額を主役とするダークなBalance Stageと、白いQuick Add Dockの非対称2領域にする。
- 空状態の見出しは巨大にせず、「最初の支出を追加」へ視線をつなぐ。
- 下段は最近の支出を主領域、月末見込みを従領域にする。
- 管理フォームはホームDOMから視覚的に分離する。

## Manage View

- ヘッダーに「ホームへ戻る」を置く。
- ページ冒頭に「管理」と当月予算を表示し、機能を説明する長文は置かない。
- 左に管理ナビ、右に選択中パネルを表示する。支出、収入、固定費、予算、分析を切り替える。
- モバイルでは管理ナビを横スクロール可能なタブにする。
- 各フォーム、履歴、削除、カテゴリ集計の既存機能と送信先を維持する。

## Interaction

- `[data-route="home"]`と`[data-route="manage"]`でビューを切り替える。
- `history.pushState`と`popstate`を使い、ブラウザの戻る操作を成立させる。
- 管理内は`[data-manage-target]`で1パネルだけ表示する。
- 「項目を指定して追加」「すべて見る」は管理ビューの支出パネルへ遷移する。
- 遷移は180〜320ms。大きなページ全体アニメーションは行わない。
- `prefers-reduced-motion`では即時切り替えにする。

## Visual System

- 白、墨色ネイビー、ニュートラルグレー、エメラルドだけを基本色とする。
- カードの均等配置を避け、面、罫線、余白、文字の階層で構成する。
- 主金額は等幅数字。補助数字は主金額と競合させない。
- 角丸は10〜20px。ピルは状態表示とモバイルタブに限定する。
- 影はBalance Stageと開いた管理パネルに限定する。

## Accessibility And Responsive

- すべての操作対象は44px以上。
- 現在ビューと現在管理パネルを`aria-current`で示す。
- ビュー切り替え後は対象見出しへフォーカスを移す。
- 390px、768px、1280px以上で横溢れを出さない。
- `hidden`要素はフォーカス対象から外れる。

## Technical Boundaries

- `routes/index.js`、`models/store.js`、`server.js`は変更しない。
- POST URL、フォーム`name`、データ構造、計算処理は変更しない。
- 新しいライブラリは追加しない。
- `package.json`には標準Nodeテストの実行コマンドだけを追加し、UI契約の回帰テストを`test/`に置く。

## Verification

- Node構文確認とHTML契約テストを行う。
- 全登録、予算更新、削除を回帰確認する。
- ホームから管理、管理内タブ、戻る導線をブラウザで確認する。
- 390pxとデスクトップ幅でスクリーンショットと横溢れを確認する。
- GitHub `main`とRender公開版の新識別子を確認する。
