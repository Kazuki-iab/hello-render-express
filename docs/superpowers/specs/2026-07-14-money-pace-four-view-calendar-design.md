# Money Pace Four-View Calendar Design

## Goal

Money Paceを「ホームと管理」の二画面から、日常利用の目的ごとに分かれた四画面へ再構成する。LINEらしさは入力体験に集中させ、金融アプリとしての精度と読みやすさを維持する。

## Chosen Information Architecture

アプリ内のhash遷移を使い、次の四画面を提供する。

- `#home`: 今月あと使える金額、今日使える目安、月末の見込み、最近の支出。
- `#input`: LINE風クイック入力、入力例、直近の支出を会話として表示。
- `#history`: 当月カレンダー、日別の支出額、選択日の支出・収入明細。
- `#manage-*`: 収入、固定費、予算、分析、詳細な支出入力と削除を管理。

デスクトップは上部ナビ、モバイルは四項目の下部ナビを使う。ブラウザの戻る・進む操作を維持する。

## Home

ホームは確認に限定する。残額を最も大きく表示し、今日と月末の見込みを補助情報として並べる。入力フォームは常設せず、「支出を追加」から入力画面へ移動する。最近の支出は最大5件を表示し、履歴画面への導線を付ける。

## LINE-Style Input

入力画面はチャットのリズムを借りるが、LINEの外観を模倣しすぎない。

- Money Pace側の短い案内を左側の白い吹き出しで表示する。
- 登録済みの直近支出を右側のエメラルド系吹き出しで表示する。
- 画面下部に入力欄と送信ボタンを一体化したコンポーザーを置く。
- 入力例は返信候補のような小さなチップとして表示する。
- 送信後は新しい吹き出し、残額更新、完了通知が順に反応する。
- 詳細入力は管理の支出パネルへ遷移させる。

## Calendar History

履歴画面は現在月のカレンダーを主役にする。

- 日付セルに支出がある場合は日別合計を表示する。
- 支出額に応じてエメラルドの濃度を三段階で変える。
- 収入がある日は小さなインジケーターを添える。
- 日付を選択すると、同じ画面の詳細領域にその日の支出と収入を表示する。
- 初期選択は今日。今日にデータがなければ、直近の取引日を選択する。
- 空の日は「この日の記録はありません」と静かに表示する。

月移動や新しい集計APIは追加しない。現行のメモリ配列から当月分だけを描画し、日付選択はクライアント側で切り替える。

## Manage

管理画面は既存の五パネルとフォーム、履歴、削除、分析を維持する。日常操作からは一段下げ、設定・詳細登録・一覧管理のためのワークスペースとして扱う。

## Visual Direction

- ホームと履歴は白と冷たいグレーを中心に、数字と罫線で精度を出す。
- 入力画面だけ背景をわずかに温度のあるグレーグリーンへ変え、会話空間であることを示す。
- LINEらしさは吹き出し、送信アイコン、返信候補、入力成功のリズムで表現する。
- LINEのロゴ、固有アイコン、ブランドカラーの完全なコピーは行わない。
- 角丸や影を増やしすぎず、チャット画面以外は現行の精密な金融UIを維持する。

## Data And API Boundaries

- `models/store.js`のデータ構造と計算処理は変更しない。
- 現在のPOST URLとフォーム`name`は変更しない。
- 新しいDB、外部ライブラリ、外部APIは追加しない。
- カレンダーとチャット表示は既存データから派生する表示専用UIとする。

## Accessibility

- すべての主要操作は44px以上にする。
- 現在の画面と選択日を`aria-current`または`aria-pressed`で通知する。
- 日付ボタンには日付、支出合計、収入有無を含むラベルを付ける。
- キーボードでページ移動、日付選択、入力、送信ができる。
- `prefers-reduced-motion`では吹き出しと画面遷移を即時表示する。

## Verification

- 四画面のHTML契約とhash遷移を自動テストする。
- クイック入力後の戻り先を`#input`に更新し、カテゴリ推定を回帰確認する。
- カレンダーの日別合計、選択日の明細、空状態を確認する。
- 390px、768px、1440pxで横溢れと固定ナビの重なりを確認する。
- 公開版でホーム、入力、履歴、管理の遷移とブラウザエラーの有無を確認する。
