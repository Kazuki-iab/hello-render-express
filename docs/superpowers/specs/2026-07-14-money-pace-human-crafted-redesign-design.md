# Money Pace Human-Crafted Redesign

## Objective

Money Paceを、カードを積み上げた生成系ダッシュボードから、数字・整列・操作のリズムで品質を感じる個人向け金融プロダクトへ再設計する。既存API、フォーム項目、データ構造、計算処理、ホームと管理の画面遷移は維持する。

## Design Diagnosis

現行画面のAIらしさは、巨大な濃色ヒーロー、左右二分のテンプレート構成、同じ角丸と影を繰り返すカード、空状態でも大きい見出し、広い余白を品質の代わりに使っている点にある。これらを削り、実用画面としての密度と編集意図を強める。

## Visual Direction

- 白と冷たいグレーを基調にし、黒に近いインク色とエメラルドだけを使う。
- 大きな面の塗り分けと強い影を廃止し、細い罫線、揃った基準線、数字の大きさで階層を作る。
- 角丸は基本8〜12px。ピル形状は状態表示だけに限定する。
- 主金額は大きくするが、周囲の余白を過剰に取らない。
- アイコンは線画に統一し、装飾目的では使わない。

## Home Composition

ホームは1枚の金融サマリーとして構成する。

1. 上部にブランド、現在月、ホーム、管理を置く。
2. 主領域の左に残額、今日の目安、月末見込み、予算進捗をまとめる。
3. 主領域の右にクイック入力を置き、入力欄を金融端末のコマンドバーのように見せる。
4. 下段は最近の支出を台帳形式で表示し、右側に収入・支出・固定費・予算消化率をコンパクトに並べる。
5. 空状態は主金額の代わりに短い案内を置き、入力欄へ自然につなぐ。

## Manage Composition

管理画面は巨大なタイトルを廃止し、コンパクトなワークスペースにする。

- 画面上部に「管理」と当月予算を同一行で置く。
- 左ナビはアイコン、名称、補助値だけの細いリストにする。
- 右パネルは影の強いカードではなく、白い作業面としてフォームと履歴を連続表示する。
- モバイルでは管理ナビを横タブにし、入力と履歴を1列で表示する。

## Interaction And Motion

- ビュー遷移は140〜220msの短いフェードと位置移動に限定する。
- クイック入力例を選ぶと入力面だけが短く反応する。
- 送信中はボタンの透明度を下げ、二重送信を防ぐ。
- 成功通知は自動で消し、エラー通知は画面遷移まで残す。
- `prefers-reduced-motion`では即時表示する。

## Accessibility And Responsive

- 操作領域は44px以上を維持する。
- フォーカスリング、`aria-current`、遷移後フォーカス、JavaScript無効時の到達性を維持する。
- 390px、768px、1280px以上で横溢れを出さない。
- 固定下部ナビがフォームや履歴の操作を隠さないよう余白を確保する。

## Technical Boundaries

- 主な変更は`controllers/appController.js`と`public/style.css`に限定する。
- 必要な表示識別子だけ`public/app.js`で扱う。
- 既存のPOST URL、フォーム`name`、モデル、計算処理は変更しない。
- 外部UIライブラリ、グラフライブラリ、フォント依存を追加しない。

## Verification

- `npm test`、Node構文確認、HTML契約確認を行う。
- 支出、収入、固定費、予算、削除、管理内の戻り先を回帰確認する。
- 実ブラウザでホーム、管理、戻る操作、通知を確認する。
- デスクトップ、390px、768pxでスクリーンショットと横溢れを確認する。
- GitHub `main`とRender公開版の反映を確認する。
